# Learning Center 🧠

Welcome to the **Learning Center**! Whether you're a seasoned Pythonista or just starting your journey, this section is designed to guide you through the core concepts and setup steps required to work on FEDGE.

We aim to explain things intuitively, using analogies and clear examples to make the "complex" feel "natural."

---

## What will you learn? 🎓

Our guides are broken down into logical steps to get you from "Zero" to "Deployable":

### 1. The Basics 🐍
Before we dive into the code, we need a stable foundation.
*   **[Virtual Environments](./venv-setup.md)**: Learn how to create an isolated playground for your project so you don't break your system Python.

### 2. Database Mastery 🗄️
Data is the heart of FEDGE. Learn how we talk to the database.
*   **[Database Setup](../database/database-setup.md)**: Understanding Engines, Sessions, and the Declarative Base.
*   **[Alembic Setup](../database/alembic-setup.md)**: Wiring up the "Time Machine" for your database schema.
*   **[Performing Migrations](../database/performing-migration.md)**: The daily workflow of evolving your data structure.

### 3. API Design (Coming Soon) 🚀
*   **Pydantic Schemas**: How we validate data coming in and going out.
*   **Dependency Injection**: How we "ask for what we need" in our routes.
*   **Service Layer**: Where the "Brain" of the application lives.

---

## How to use these docs? 📖

1.  **Follow the Order**: If you're new, start from the top. Each guide builds on the previous one.
2.  **Copy-Paste Friendly**: We provide complete code blocks that you can use immediately.
3.  **Look for Emojis**: We use them to highlight key concepts and warnings.
